基于您提供的新闻源，这是一个非常专业的**金融监管与宏观数据聚合平台**。我为您规划一个系统化的前端展示方案，兼顾专业性、易用性和视觉清晰度。

## **一、项目命名**

*   **中文名称：** **金源观潮**

*   **英文名称：** **PolicyPulse**
    *   寓意：捕捉政策与市场的脉搏。简洁、现代、国际化。
*   **标语：** *From Official Sources, For Smarter Decisions.*
    （源于官方，智见决策）

## **二、整体视觉与交互规划**

### **核心理念**
*   **专业权威 (Professional & Authoritative):** 采用金融、数据类网站常见的深色系、蓝色系，营造稳重、可信赖的氛围。
*   **清晰高效 (Clear & Efficient):** 信息密度高但布局清晰，支持快速扫描、筛选和定位关键信息。减少装饰性元素。
*   **数据驱动 (Data-Driven):** 将纯文本公告与可视化图表（时间序列、数据点）紧密结合。
*   **差异化 (Differentiated):** 通过“发布日历”、“数据快照”等功能，与传统新闻网站区分开。

### **视觉风格**
*   **主色调:**
    *   **主色:** 深蓝色 (#0F1C3F) 或藏青色 (#1A2B4C) - 象征专业、稳定。
    *   **辅助色/强调色:** 科技蓝 (#1890FF)、金融金 (#D4AF37) 或冷静绿 (#00A876) - 用于按钮、高亮、图表。
    *   **背景色:** 浅灰色 (#F5F7FA) 或白色 (#FFFFFF)。
    *   **文字色:** 深灰 (#333333)、中灰 (#666666)、浅灰 (#999999)。
*   **字体:**
    *   **英文:** Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto (无衬线字体，清晰易读)。
    *   **中文:** 阿里巴巴普惠体、思源黑体、苹方/PingFang SC (同样选择现代的无衬线字体)。
*   **设计规范:** 采用卡片式设计，阴影柔和，留白充足，确保信息层级分明。

## **三、网站结构与页面规划**

### **1. 首页 / Dashboard**
*   **设计：** 仪表盘式布局，一眼览尽全局动态。
*   **核心模块：**
    *   **数据发布日历 (Calendar):** 可视化日历，高亮标记未来24小时/一周内即将发布的**中/美关键数据**（如中国CPI、美国非农）。点击可查看详情。
    *   **全球要闻快讯 (Headline Ticker):** 顶部跑马灯或滚动栏，实时显示最新、最重要的5-10条公告标题（如“美联储宣布利率决议”、“证监会发布新规”）。
    *   **核心指标看板 (Key Metrics):**
        *   **中国:** 实时展示最新MLF利率、LPR、10年期国债收益率（如果数据源支持）。
        *   **美国:** 实时展示联邦基金利率目标区间、10年期美债收益率、最新CPI同比。
        *   (数据通过FRED、中国货币网等API获取并缓存)。
    *   **最新源流 (Latest Stream):** 按时间倒序列出所有抓取源的最新条目（可混合或按区域分Tab），每条显示：**源图标、标题、发布时间（精确到分钟）、简短摘要/关键词**。
    *   **热门关注 (Trending):** 根据用户点击/收藏量，列出今日最受关注的公告或政策文件。

### **2. 源分类导航页 (Sources)**
*   **设计：** 左侧为固定导航栏（类似管理后台），右侧为主要内容区。
*   **左侧导航结构 (树状+图标):**
    ```
    📈 中国官方
    ├─ 中国人民银行 (PBoC)
    ├─ 国家统计局 (NBS)
    ├─ 证监会 (CSRC)
    └─ ...
    🏛️ 美国/全球官方
    ├─ 美联储 (Fed)
    ├─ 美国财政部 (U.S. Treasury)
    └─ ...
    📄 市场披露
    ├─ 上交所 (SSE)
    ├─ 深交所 (SZSE)
    └─ 巨潮资讯 (CNINFO)
    ⛽ 大宗商品
    ├─ EIA
    └─ IEA
    ```
*   **功能：** 点击任一源，右侧主区域展示该源的专属页面。

### **3. 单个源详情页 (例：中国人民银行)**
*   **页头：** 源Logo、名称、简介、最后抓取时间。
*   **主体：**
    *   **筛选器 (Filter):** 按日期范围、栏目（如“货币政策报告”、“公开市场操作公告”）、关键词进行筛选。
    *   **内容列表 (Content List):**
        *   每条显示：**标题、所属栏目、发布时间、操作按钮**。
        *   **标题样式:** 重要报告（如货币政策报告）可加粗或使用强调色。
        *   **操作按钮:** `预览`、`查看原文`（新标签页打开官网）、`收藏`、`分享`。
    *   **如果该源有可获取的时间序列数据 (如FRED， BLS):**
        *   在页面上方提供**数据图表模块**，内嵌小型交互图表，用户可选择指标、时间范围查看。

### **4. 公告/文件详情页**
*   **设计：** 阅读器模式，专注内容。
*   **布局：**
    *   顶部面包屑导航、标题、元信息（来源、发布时间、栏目）。
    *   **正文区域:**
        *   **智能提取视图 (默认):** 使用NLP技术尝试提取核心段落、关键数据表格，并进行**格式化展示**（如将表格转化为更清晰的HTML表格）。
        *   **原文视图:** 提供切换按钮，展示原始抓取的HTML或纯文本，保持官方原文格式。
        *   **PDF预览:** 如果原文是PDF，内嵌PDF.js预览器。
    *   **侧边栏/底部工具栏:**
        *   **关键标签/实体识别:** 自动标出提到的机构、人物、关键指标（如“利率”、“GDP”）。
        *   **相关链接:** 链接到同一主题的其他文件或数据。
        *   **下载按钮** (用于PDF/Excel)。
        *   **收藏/笔记功能。**

### **5. 数据探索页 (Data Explorer) - 针对API数据源 (如FRED, EIA)**
*   **设计：** 类似金融数据终端（如Bloomberg）的简约版。
*   **功能：**
    *   **指标分类浏览:** 按国家、类别（利率、通胀、就业）浏览。
    *   **图表绘制工具:** 用户选择1-N个指标，生成自定义图表（折线图、柱状图），可调整时间范围、频率。
    *   **数据表格:** 提供图表对应数据的下载（CSV格式）。

### **6. 全局搜索**
*   **功能：**
    *   支持全文搜索（标题、正文）。
    *   高级筛选：按来源、时间、内容类型（新闻、报告、数据）筛选。
    *   搜索结果高亮关键词。

## **四、关键特性与组件**

1.  **时间线视图 (Timeline View):** 以时间轴方式展示某个主题（如“美联储加息周期”）相关的所有公告、报告、数据点，形成叙事线。
2.  **我的收藏夹/观察列表 (Watchlist):** 用户可收藏特定源、栏目或关键词，在个人中心集中查看更新。
3.  **通知/提醒 (Alerts):**
    *   系统级：可订阅特定源或关键词，通过站内信或邮件接收新内容提醒。
    *   数据级：对FRED等数据指标设置阈值提醒（如“当美国CPI>5%时通知我”）。
4.  **夜间模式 (Dark Mode):** 必备功能，适合长时间分析。
5.  **移动端适配 (Mobile Responsive):** 信息流优先，简化筛选器，确保在小屏幕上也能高效阅读。

## **五、技术栈建议 (前端)**

*   **框架:** **React** (生态丰富) 或 **Vue 3** (上手快)，配合 TypeScript 保证代码质量。
*   **UI组件库:**
    *   **Ant Design** (React) 或 **Element Plus** (Vue)：提供大量专业、企业级组件，快速搭建后台类应用。
    *   如果追求更轻量、自定义，可选 **Tailwind CSS**。
*   **图表库:** **ECharts** 或 **AntV** (国内，文档友好)，功能强大，能满足复杂的金融图表需求。
*   **状态管理:** Redux Toolkit (React) 或 Pinia (Vue)。
*   **路由:** React Router 或 Vue Router。
*   **构建工具:** Vite (开发体验极佳)。

## **总结**

您要构建的是一个服务于**专业投资者、分析师、研究员的“一级信息枢纽”**。前端设计的成功关键在于：
**将杂乱无章的官方原始信息，通过清晰的分类、直观的视觉、高效的工具，转化为可快速消费、可深度分析的结构化知识。**

建议采用 **“渐进式”开发**：
1.  **第一阶段 (MVP):** 实现核心源（如人行、美联储、巨潮）的列表与详情页，以及基础搜索和筛选。
2.  **第二阶段:** 加入数据可视化（FRED API集成）、发布日历、用户个人中心。
3.  **第三阶段:** 开发高级功能如智能提取、时间线、自定义提醒等。

这个规划为您提供了一个兼具专业性与可用性的蓝图，您可以根据实际开发资源进行调整。
